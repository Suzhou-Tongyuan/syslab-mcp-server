# Syslab MCP 接口与参数说明

本文档用于汇总当前 `syslab-mcp-server` 的接口、启动参数及关键行为约定。

## 1. 服务概览

当前服务名：

- `syslab-mcp-server`

MCP 协议入口能力：

- `initialize`
- `notifications/initialized`
- `ping`
- `tools/list`
- `tools/call`

核心执行模式：

- Go 进程负责 MCP 协议解析、参数管理、帮助文档索引和会话管理
- `nodesktop` 模式下，Julia 侧通过内嵌桥接脚本启动，Go 与 Julia 通过标准输入/输出进行协议通信
- `desktop` 模式下，Go 启动 Syslab 桌面进程，并通过本地 IPC 与 Syslab API 通信

主要实现位置：

- 启动入口：`cmd/syslab-mcp-core-server/main.go`
- 工具注册：`pkg/tools/catalog.go`
- Julia 会话管理：`internal/session/manager.go`
- Ty 文档索引：`internal/tydocs/catalog.go`
- 环境发现：`internal/tydocs/discovery.go`
- 启动脚本和环境解析：`internal/syslabenv/env.go`

## 2. 启动参数

### 2.1 `--syslab-launcher`

用途：

- 指定 `julia-ty.bat` 或 `julia-ty.sh` 的绝对路径

说明：

- 如未显式传入，程序会结合 `--julia-root/bin` 自动解析

### 2.2 `--syslab-root`

用途：

- 指定 Syslab 安装根目录
- 必填参数

主要用途：

- 读取 Syslab 版本信息
- 定位默认帮助文档根目录
- 定位默认帮助文档索引文件 `Tools/AIAssets/tydocs-index.json`

### 2.3 `--help-docs-root`

用途：

- 指定帮助文档 `projects` 根目录

说明：

- 可选参数
- 如果提供，Ty 文档索引优先从该目录读取
- 如果不提供，默认使用：
  `<syslab-root>/Tools/AIAssets/syslabHelpSourceCode/projects`

### 2.4 `--julia-root`

用途：

- 指定 Julia 安装根目录

说明：

- 选填参数
- 不传时，优先从 `syslab-env.ini` 中读取 `JULIA_HOME`
- Linux 下如果 `syslab-env.ini` 未提供，再自动从 `<syslab-root>/Tools` 下查找 `julia-*` 目录

说明：

- 用于辅助解析 `julia-ty.bat` 或 `julia-ty.sh`

### 2.5 `--mlang-path`

用途：

- 预留参数

当前状态：

- 未参与核心逻辑

### 2.6 `--initialize-syslab-on-startup`

用途：

- 在 MCP `initialize` 阶段预启动 Julia 会话

当前状态：

- 默认值为 `false`
- 如果为 `false`，则在首次调用 Julia 相关工具时懒启动

### 2.7 `--initial-working-folder`

用途：

- 指定工具执行时的初始工作目录

说明：

- 不传时继承当前进程工作目录

### 2.8 `--syslab-display-mode`

用途：

- 执行模式参数

当前状态：

- 默认值为 `nodesktop`
- `nodesktop`：仅使用 Julia bridge，不启动 Syslab 桌面
- `desktop`：启动 Syslab 桌面，通过本地 IPC 调用 Julia REPL 相关 API

### 2.9 `--pkg-offline`

用途：

- 控制启动的 Julia 会话是否默认启用离线包模式

当前状态：

- 默认值为 `true`
- 启动 bridge 进程时会注入环境变量 `JULIA_PKG_OFFLINE=true`
- 传入 `--pkg-offline=false` 时会注入 `JULIA_PKG_OFFLINE=false`

### 2.10 `--log-level`

用途：

- 日志级别

### 2.11 `--disable-telemetry`

用途：

- 预留参数

当前状态：

- 默认值为 `true`

### 2.12 `--desktop-startup-timeout`

用途：

- `desktop` 模式下等待 Syslab 桌面进程连回 MCP 的超时

当前状态：

- 默认值为 `2m`

### 2.13 `--desktop-ready-timeout`

用途：

- `desktop` 模式下等待 `syslab.hasStarted` ready 握手的超时

当前状态：

- 默认值为 `5m`

### 2.14 `--desktop-attach-timeout`

用途：

- `desktop` 模式下附着已有 IPC 会话的超时

当前状态：

- 默认值为 `15s`

### 2.15 `--desktop-repl-timeout`

用途：

- `desktop` 模式下启动或停止 Julia REPL 的超时

当前状态：

- 默认值为 `2m`

### 2.16 `--desktop-control-timeout`

用途：

- `desktop` 模式下 `openFolder`、`openFile`、获取终端输出等控制命令的超时

当前状态：

- 默认值为 `30s`

## 3. MCP 工具接口

### 3.1 `detect_syslab_toolboxes`

用途：

- 返回 Syslab 版本信息
- 返回 Julia depot 路径相关信息
- 返回 Syslab Julia 环境中已安装的 Julia 包信息，包括 `Ty*` 包
- 返回帮助文档发现结果

输入参数：

- `include_all_packages`：可选，布尔值。默认 `false`。不填或为 `false` 时，仅返回包名以 `Ty` 开头的包；为 `true` 时，返回当前 Julia 全局环境中的所有包

返回关键字段：

- `tool`
- `syslab_env_file`
- `julia_launcher_file`
- `syslab_version_file`
- `syslab_version`
- `julia_home`
- `julia_depot_path`
- `julia_plugin_path`
- `ty_conda3`
- `python`
- `syslab_launcher`
- `session_pid`
- `julia_version`
- `bindir`
- `depot_path`
- `active_project`
- `manifest_file`
- `session_reuse`
- `toolboxes`

### 3.2 `evaluate_julia_code`

用途：

- 在全局 Julia 环境中执行一段 Julia 代码
- `desktop` 模式下，通过 Syslab 桌面中的 Julia REPL 执行

输入参数：

- `code`：必填，Julia 代码字符串

### 3.3 `run_julia_file`

用途：

- 执行指定的 `.jl` 文件

输入参数：

- `script_path`：必填，Julia 脚本路径

说明：

- 使用脚本所在目录作为工作目录
- 仍固定运行在全局 Julia session 中
- `desktop` 模式下，先打开指定文件，再执行当前活动文件

### 3.4 `restart_julia`

用途：

- 重启全局 Julia 会话
- `desktop` 模式下，仅重启 Julia REPL，不关闭 Syslab 桌面进程

输入参数：

- `working_directory`：可选，字符串。重启后全局 Julia session 的工作目录。不填时保持当前行为

### 3.5 `search_syslab_docs`

用途：

- 搜索 Syslab Julia 环境中的本地包文档

输入参数：

- `query`：必填，搜索关键词
- `package`：可选，包名过滤
- `max_results`：可选，最大返回条数

### 3.7 `read_syslab_doc`

用途：

- 读取单个已索引帮助文档

输入参数：

- `doc_path`：必填，文档绝对路径

### 3.8 `map_matlab_functions_to_julia`

用途：

- 将一组 MATLAB 函数名映射到 Syslab Julia 环境中的候选函数
- 返回候选函数摘要和本地帮助文档路径
- 用于 MATLAB 代码迁移、函数替换和 MATLAB 到 Syslab Julia 的映射查询
- 不等同于对 Syslab 帮助文档的直接搜索

输入参数：

- `symbols`：必填，MATLAB 函数名字符串数组
- `max_results_per_symbol`：可选，每个 MATLAB 函数最多返回的候选数量，默认 `3`

返回关键字段：

- `resolved`
- `resolved[].matlab_symbol`
- `resolved[].candidates`
- `resolved[].candidates[].syslab_symbol`
- `resolved[].candidates[].package`
- `resolved[].candidates[].summary`
- `resolved[].candidates[].doc_path`
- `resolved[].candidates[].source`
- `unresolved`

说明：

- 该接口的查询入口是 MATLAB 函数名
- 返回的是 Syslab Julia 环境中的候选函数及其文档，不是 MATLAB 原始帮助文档
- 如果要搜索 Syslab Julia 包文档正文，仍应使用 `search_syslab_docs`

### 3.9 `read_syslab_skill`

用途：

- 读取 Syslab skill markdown 文件内容
- 传入 `skill_path=default` 时，读取服务启动时解析并在 `initialize` 阶段使用的默认 `SKILL.md`
- 传入绝对路径时，读取指定路径的 markdown 文件
- 服务端仍兼容省略 `skill_path` 时读取默认 skill 的旧行为，但工具 schema 会将该参数声明为必填，以便不支持无参调用的客户端稳定调用

输入参数：

- `skill_path`：必填，字符串。传入 `default` 时，读取服务启动时解析出的默认 skill 文件；传入绝对路径时，读取指定 markdown 文件

返回关键字段：

- `tool`
- `skill_path`
- `loaded`
- `content`
- `truncated`

说明：

- 当 `skill_path` 省略时，返回的 `skill_path` 与服务初始化实际使用的 skill 文件路径一致
- 当文件不存在时，返回 `loaded=false` 与空 `content`，不视为接口异常
- 当内容超过服务端截断阈值时，返回 `truncated=true`

## 4. 关键实现行为

### 4.1 Julia 桥接脚本

当前不再依赖仓库中的 `bridge/` 目录。

现状：

- Julia 桥接脚本内嵌在可执行文件中
- 启动时提取到本地缓存目录
- 然后由 `julia-ty.bat` 或 `julia-ty.sh` 实际执行

### 4.2 Julia 会话启动与健康检查

会话创建流程：

1. 校验 `syslab-launcher` 与桥接脚本存在
2. 启动 Julia 桥接进程
3. 发送 `health` 请求
4. 读取桥接响应
5. 成功后将该进程纳入会话管理

`desktop` 模式会话创建流程：

1. 创建本地 IPC 端点
   - Windows：Named Pipe
   - Linux：Unix Domain Socket
2. 启动 Syslab 桌面进程，并注入环境变量 `SYSLAB_TEST_API_PIPE_NAME`
3. 等待 Syslab 客户端连接
4. 等待就绪消息 `syslab.hasStarted=True`
5. 成功后将该桌面进程纳入会话管理

### 4.3 `JULIA_DEPOT_PATH` 取值优先级

当前逻辑做了三层取值：

1. 优先读取 `~/.syslab/syslab-env.ini`
2. 如果没有，再从 `julia-ty.bat` 或 `julia-ty.sh` 中解析
3. 如果仍未找到，则读取进程环境变量 `JULIA_DEPOT_PATH`

环境变量回退规则：

- Windows 按 `;` 拆分，Unix 按 `:` 拆分
- 按顺序检查每一段路径
- 返回第一个实际存在且为目录的 depot 路径
- 如果所有候选路径都不存在，则视为未找到

### 4.4 Syslab 版本读取逻辑

当前从：

- `<syslab-root>/versionInfo/build_info.json`

读取版本信息。

### 4.5 文档检索与 MATLAB 映射检索的区分

当前有两类不同入口：

- `search_syslab_docs`
  - 面向 Syslab Julia 环境中的本地包文档搜索
  - 输入是文档关键词
  - 返回匹配到的本地帮助文档条目

- `map_matlab_functions_to_julia`
  - 面向 MATLAB 代码迁移和 MATLAB 到 Syslab Julia 的函数映射查询
  - 输入是 MATLAB 函数名列表
  - 返回对应的 Syslab Julia 候选函数摘要和文档路径

推荐工作流：

1. 客户端先从 MATLAB 代码中提取候选函数名
2. 调用 `map_matlab_functions_to_julia` 获取候选映射
3. 仅对少数关键候选调用 `read_syslab_doc`
4. 如需更广泛的 Syslab Julia 包文档搜索，再调用 `search_syslab_docs`

### 4.5 帮助文档索引逻辑

默认帮助文档索引文件：

- `<syslab-root>/Tools/AIAssets/tydocs-index.json`

`initialize` 阶段行为：

- 只尝试读取该索引文件
- 如果索引文件不存在、损坏，或者 `AIAssets` 文件夹不存在，服务仍可正常启动

独立索引生成脚本：

- `scripts/build_tydocs_index.py`

输入输出约定：

- 输入：`AIAssets` 文件夹
- 输出：`tydocs-index.json`

索引路径布局兼容：

- 新结构：`projects/` 与 `static/FunctionTable/`
- 旧结构：`syslabHelpSourceCode/projects/` 与 `SearchCenter/static/FunctionTable/`

## 5. 汇报建议

面向汇报场景，建议重点突出：

- 当前 MCP 已形成“参数配置 + 会话管理 + 文档索引 + Julia 执行”的完整闭环
- 帮助文档索引支持安装包预制，避免启动阶段现场构建
- Julia 环境路径获取已做多层兜底，兼容不同部署结构
- 桥接脚本已内嵌，部署时不再依赖仓库源码目录
